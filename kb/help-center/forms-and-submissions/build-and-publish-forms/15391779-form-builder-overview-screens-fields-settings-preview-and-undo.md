## Article
Form Builder is where you design the visitor experience, configure tracking, and publish the version that collects submissions.

## What you see in the top bar

- **Back arrow**: return to **Dashboard** > **Forms**.
- **Form name and status**: confirm whether you are editing a draft, published form, or archived form.
- **Theme toggle**: switch the dashboard display between light and dark mode.
- **Undo / Redo**: reverse or reapply recent builder changes.
- **Preview**: open the public form URL in a new tab.
- **Publish / Save**: publish a new form or save changes to an already published form.

## Use the left sidebar

- **Screens**: add steps, choose the active screen, duplicate screens, hide screens, move screens, or remove screens.
- **Form fields**: search and add field types to the selected screen.
- **Form settings**: configure title, design, submissions, tracking, integrations, and redirect behavior.
- **Field settings**: select a field to configure type, required state, options, defaults, validation, scoring, and logic.
- **Screen settings**: select a screen to edit title, description, visibility, and button labels.

## Use the canvas

The canvas shows each screen in order. You can edit screen titles, screen descriptions, field labels, helper text, options, and default values directly from the canvas.

## A simple first workflow

1. Create or open a form from **Dashboard** > **Forms**.
2. Add the screens you need.
3. Add fields to each screen.
4. Open **Form settings** to set design, submission handling, tracking, and redirect behavior.
5. Select **Publish** or **Save**.
6. Preview the form and submit a test response.

> Best practice: build the screen flow first, then tune field settings, styling, and tracking.

## Troubleshooting

**The Save button is disabled.**

- For a published form, the button only becomes active after you make a new change.

**I selected the wrong item.**

- Close the settings panel or select another screen or field in the canvas.
