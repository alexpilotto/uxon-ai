## Article
Testing Plan settings let you align your workflow stages, test categories, and visible columns to the way your team operates.

## Open Testing Plan settings

1. Go to **Testing Plan**.
2. Select the **Settings** icon in the header.
3. Use **General** and **Advanced** tabs to configure your setup.

## Update statuses

- Rename custom statuses.
- Reorder statuses using drag handles.
- Change status colors.
- Add new statuses with **Add status**.
- Delete custom statuses you no longer need.

System statuses **Running**, **Paused**, and **Ended** are fixed and cannot be deleted.

## Update test types

- Rename or reorder test types.
- Add new test types.
- Delete obsolete test types.

## Control optional columns (Advanced tab)

- **Enable expected uplift / KPI**
- **Enable relationships**
- **Show statistical significance** (available when relationships are enabled)
- **Enable learnings**

## Important guardrails

You cannot remove a status or test type that is currently used by existing tests. Reassign those tests first, then update settings.

## Save settings

Select **Save changes** to apply updates across your Testing Plan table and test forms.
