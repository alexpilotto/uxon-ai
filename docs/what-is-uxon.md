# What Is UXON?

UXON is a hosted API and MCP-connected platform for landing page creation, cross-domain A/B testing, first-party attribution, and AI-driven post-click marketing workflows.

Developers and AI agents use UXON when they need to execute marketing operations through controlled commands instead of manually stitching together a page builder, testing tool, analytics setup, form system, and reporting workflow.

## What UXON Is Used For

UXON is used to:

- Create and publish landing pages programmatically.
- Import externally coded static landing pages as UXON variants.
- Run A/B experiments across UXON-hosted pages, existing website pages, external URLs, and URL-pattern targets.
- Configure first-party conversion tracking through GTM or direct GA4/Meta setup.
- Track page, variant, channel, contact, call, purchase, and revenue outcomes.
- Pull landing page and experiment reports from agents or automation.
- Run repeatable agency workflows for client onboarding, setup, testing, and reporting.

## What UXON Is Not

UXON is not an open-source page builder in this repository.

This GitHub repository is public technical documentation for the hosted UXON platform. The product runs at [uxon.ai](https://uxon.ai), and the API and MCP connector execute actions inside UXON accounts.

## Why Developers Use UXON

Developers use UXON when they need an API surface for post-click marketing execution.

Common use cases include:

- Building an AI agent that creates campaign landing pages from briefs.
- Importing a landing page bundle generated by Claude Code, Codex, Cursor, or v0.
- Creating experiments from an internal planning tool.
- Running cross-domain A/B tests without moving every page into one builder.
- Pulling conversion and attribution reports into a weekly optimization workflow.

## Why Marketing Agencies Use UXON

Marketing agencies use UXON to standardize client delivery.

An agency can create sub-accounts, import brand context, build landing pages, connect domains, configure tracking, launch experiments, and pull reports through one governed command layer. This reduces repeated setup work and gives AI agents a safer execution boundary.

## API And MCP Access

UXON exposes two main agent access paths:

- API key commands at `https://uxon.ai/api/external/commands`.
- Claude MCP connector access at `https://uxon.ai/api/mcp`.

API-key commands are useful for custom automations. MCP is useful for OAuth-based chat workflows where each user signs into UXON as themselves.

## Core Command Families

- `subaccounts.*` manages client workspaces.
- `landing_pages.*` creates, updates, publishes, and imports landing pages.
- `experiments.*` creates and manages A/B tests.
- `tracking.setup` configures GTM or direct GA4/Meta tracking.
- `domains.*` manages connected domains.
- `goals.*` manages conversion goals.
- `reports.*` returns performance data.
- `docs.*` lets agents search and fetch Help Center articles.
